0. 사전 정의: 입력 데이터 구조

이 문서의 모든 시스템은 설문 결과로부터 도출된 8축 성향 카운트(axes)를 입력으로 받습니다.

0-1. axes 객체

// 각 사용자별 1개씩 생성됨
axes = {
  개방성: 0,    // 새로운 경험, 호기심, 창의성
  성실성: 0,    // 계획, 책임감, 목표 달성 능력

각 설문 문항의 선택지에 #개방성, #성실성 등의 태그가 매핑되어 있으며, 사용자가 선택한 옵션의 태그를 축(axis)으로 변환하여 1씩 카운트합니다.

0-2. SLOT_COUNT — 이게 뭐고 왜 필요한가?

핵심 개념: 24개 설문 문항 전체에서 각 축(axis)에 매핑되는 선택지의 총 개수가 SLOT_COUNT입니다.

예를 들어 성실성의 SLOT_COUNT가 22이라는 건, 24문항 전체를 통틀어 #성실성으로 매핑되는 선택지가 총 22개 존재한다는 뜻입니다. 한 사람이 모든 문항에서 성실성 태그가 붙은 옵션만 골랐다면 최대 22점이 나옵니다.

왜 필요한가? — 축 간 공정 비교를 위해서입니다.

애착불안은 슬롯이 6개밖에 안 되고 성실성은 22개나 되므로, 단순 카운트(raw count)로 비교하면 성실성이 항상 높게 나옵니다. 그래서 지배적 성향(dominantTrait)을 결정할 때 Z-score 정규화로 공정하게 비교합니다. (자세한 내용은 Section 0-6 참조)

// Z-score 정규화 예시 (v4.0)
z_성실성  = (6 - 5.50) / 1.9039  →  z = 0.26
z_애착불안 = (4 - 1.50) / 1.0607  →  z = 2.36

// → raw count는 성실성(6)이 높지만,
//   Z-score는 애착불안(2.36)이 월등히 높음 → 지배적 성향 = 애착불안
//   "평균 대비 얼마나 극단적인가"를 공정하게 비교

0-3. 지배적 성향(Dominant Trait) 결정

// v4.0: Z-score 정규화로 변경
function getDominantTrait(axes) {
    let maxZ = -Infinity, dom = '우호성';
    for (const [trait, count] of Object.entries(axes)) {
        const z = (count - AXIS_MEAN[trait]) / AXIS_STD[trait];
        if (z > maxZ) { maxZ = z; dom = trait; }
    }
    return dom;
}

이 지배적 성향이 파동 색상, 매칭 타입, 결 해석 프로필 전부에 핵심 입력으로 사용됩니다.

0-8. 24문항 전체 목록 — 질문 · 선택지 · 태그 · 축 매핑

이 섹션의 목적: 프론트엔드 개발자가 axes 카운트가 어떻게 산출되는지 원본 데이터를 보고 직접 검증할 수 있게 합니다.
24문항 × 4선택지 = 96개 태그가 8축에 매핑되며, 사용자가 선택한 옵션의 태그를 해당 축에 1씩 카운트합니다.

// 사용자가 Q1에서 3번("환경이 바뀌어도 적응하려고 노력하는 편이에요")을 선택하면:
// → 태그: #개방성 → axes['개방성']++

// 24문항 전부 이렇게 누적하면 최종 axes 완성

function getDominantTrait(axes) {
    let maxZ = -Infinity, dom = '우호성';
    for (const [trait, count] of Object.entries(axes)) {
        const z = (count - AXIS_MEAN[trait]) / AXIS_STD[trait];
        if (z > maxZ) { maxZ = z; dom = trait; }
    }
    return dom;
}

0-7. 슬롯 불균형은 문제인가?

결론: 슬롯 불균형 자체는 문제가 아닙니다.

이유 1: 설계상 의도적 구조

24문항은 삼중뇌 이론(파충류뇌/포유류뇌/영장류뇌) 기반으로 설계되었습니다. Big5 성격 요인은 생활습관, 에너지, 가치관 등 광범위한 영역을 측정하므로 자연스럽게 많은 문항에 태그가 분포됩니다(5축 합계 70개, 72.9%). 반면 애착 유형은 연락/부재/싸움/화해 등 특정 관계 상황에서만 측정 가능하므로 슬롯이 적습니다(3축 합계 26개, 27.1%).

이유 2: 심리 측정 분야의 표준 관행

NEO-PI-R 등 표준화된 Big5 검사도구에서도 하위척도별 문항 수가 다릅니다. 중요한 것은 문항 수 자체가 아니라 "다른 문항 수를 가진 척도를 공정하게 비교하는 정규화 방법"입니다.

이유 3: Z-score가 불균형을 완전히 해소

Z-score는 각 축의 이론적 평균(μ)과 표준편차(σ)를 사용하여 "이 사람의 이 점수가 얼마나 비정상적으로 높은가"를 축 간 동일한 척도로 변환합니다. 슬롯이 6개든 22개든, Z-score의 기대 평균은 0, 분산은 1로 동일합니다.

// 슬롯 불균형의 영향
//   count/SLOT: E[norm]=0.25 (동일), Var[norm]=0.25×0.75/n (다름!) ← 문제
//   Z-score:    E[z]=0 (동일),       Var[z]=1 (동일)                ← 해결

결론: 슬롯 수를 인위적으로 맞출 필요 없이, Z-score 정규화만으로 공정성이 보장됩니다. 오히려 슬롯 수를 억지로 맞추면 설문의 심리학적 타당성이 훼손됩니다.

1. 파동 표현 (Wave Visualization)

Canvas 위에 사인(sin) 파형을 실시간 애니메이션으로 렌더링합니다. 남녀 각각 별도 파형으로 표시되며, 3축(진폭/주파수/위상)이 성격을 시각적으로 표현합니다.

1-1. 3축 계산 공식

① 진폭 (Amplitude) — 관계 에너지 크기

파동의 높이. 높을수록 관계에 적극적이고 에너지가 강합니다.

raw = 외향성 × 1.2 + 개방성 × 0.6 + 애착안정 × 0.4 - 애착회피 × 0.8
level = clamp(round(raw + 0), 1, 10)           // offset = 0
pixelAmp = 8 + (level - 1) / 9 × (maxAmplitude - 8)
// maxAmplitude = (canvasHeight / 2) - 60

② 주파수 (Frequency) — 감정 리듬

파동이 얼마나 빨리 반복되는지. 높을수록 감정 변화가 빠르고 변동적입니다.

raw = 애착불안 × 1.0 + 개방성 × 0.4 - 정서안정 × 0.8 - 성실성 × 0.3
level = clamp(round(raw + 6), 1, 10)           // offset = 6
pixelFreq = 0.008 + (level - 1) / 9 × 0.027

③ 위상 (Phase) — 리드/팔로우 역할

두 파동 간의 시간차. 차이가 클수록 한 사람이 이끌고 한 사람이 따르는 구조입니다.

raw = 외향성 × 0.8 + 애착안정 × 0.5 - 우호성 × 0.9 - 애착회피 × 0.7
level = clamp(round(raw + 5.5), 1, 10)         // offset = 5.5
phaseDiff = (mPhaseLevel - fPhaseLevel) × 0.3  // 남녀 위상차

function getDominantTrait(axes) {
    let maxZ = -Infinity, dom = '우호성';
    for (const [trait, count] of Object.entries(axes)) {
        const z = (count - AXIS_MEAN[trait]) / AXIS_STD[trait];
        if (z > maxZ) { maxZ = z; dom = trait; }
    }
    return dom;
}

3. 결 해석 (Gyeol Interpretation)

각 개인의 성격 프로필 + 커플 종합 해석을 텍스트로 생성합니다.

3-1. 개인 프로필 생성 흐름

Step 1) 모든 8축을 Z-score 정규화하여 내림차순 정렬 → top1, top2 결정
Step 2) 3축 레벨(진폭/주파수/위상) 계산 → 사분면(Quadrant) 결정
        - ampLevel >= 6 → 에너지높음(H), < 6 → 낮음(L)
        - freqLevel <= 5 → 안정적(H),   > 5 → 변동적(L)
        - 조합: HH / HL / LH / LL
Step 3) profiles[top1+top2][quad] → 프로필 한 줄 설명
Step 4) 3축 레벨별 10단계 설명 + 색상 해석 조합
